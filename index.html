[01]  SOBREABOUT

A ponte entre Engenharia de Dados pesada e decisão estratégica. The bridge between heavy-lifting data engineering and strategic decisions.

Trabalho no ponto onde engenharia de dados encontra decisão de negócio. Desenhei Data Warehouses corporativos, orquestrei pipelines em produção e construí camadas semânticas que sustentam o ciclo de finanças, comercial e operações.

Mais do que criar dashboards, resolvo problemas arquiteturais na raiz. Isso inclui eliminar anomalias estatísticas, como fan-outs e produtos cartesianos, e otimizar custos de queries (FinOps) em ambientes Cloud.

Um ambiente analítico só funciona de verdade quando finanças, comercial e operações olham para o mesmo painel e enxergam o mesmo número, sem planilhas paralelas. O que eu entrego não é apenas visualização de dados, mas a infraestrutura de confiança que sustenta a tomada de decisão.

I work where heavy data engineering meets business decisions. I've designed corporate Data Warehouses, run pipelines in production, and built semantic layers that finance, sales, and operations rely on every closing cycle.

Rather than just building dashboards, I solve root architectural problems. This ranges from eliminating statistical anomalies, like fan-outs and Cartesian products, to optimizing query costs (FinOps) in Cloud environments.

An analytical environment only works when finance, sales, and operations look at the same dashboard and see the exact same number, without parallel spreadsheets. I don't just deliver data visualization. I build the reliable infrastructure that makes strategic decisions possible.

Data Warehouse Corporativo & Visão 360º Financeira Corporate Data Warehouse & Financial 360º View

[02]   thecoffee.looker.case_02_fanout_resolution

Camada Semântica em LookML & eliminação de fan-out LookML Semantic Layer & fan-out elimination

// problema// problem

Múltiplas tabelas-fato (vendas, produtos, financeiro) compartilhando dimensões em granularidades diferentes. Joins ingênuos geravam produtos cartesianos silenciosos — receita inflada, ticket médio distorcido e divergência entre áreas no mesmo período.

Multiple fact tables (sales, products, finance) sharing dimensions at different grains. Naive joins produced silent Cartesian products — inflated revenue, distorted ticket size, and disagreement between teams on the same period.

// solução// solution

Arquitetura completa da camada semântica em LookML, com symmetric aggregates nos joins many-to-many críticos. Layer de measures certificadas (# CERTIFIED) versionadas e protegidas por PR review.

Full LookML semantic-layer architecture with symmetric aggregates on the critical many-to-many joins. Certified measures (# CERTIFIED) versioned and protected by PR review.

// impacto// impact

Convergência de KPIs entre finanças, comercial e operações em ciclo único de fechamento. Eliminação dos relatórios paralelos em Excel. O LookML virou referência interna — documentação viva da operação.

Convergent KPIs across finance, sales, and ops in a single closing cycle. Parallel Excel reports gone. The LookML became the internal reference — living documentation of the operation.

[03]   thecoffee.dataops.case_03_dataform_migration
orquestração  +  dataopsorchestration  +  dataops 2026

Pipeline as Code & migração para Dataform Pipeline as Code & Dataform migration

// problema// problem

Pipeline rodando em BigQuery Scheduled Queries — sem versionamento, sem DAG explícito, sem PR review. Mudança numa tabela quebrava outra silenciosamente. Tabelas em full-refresh diário sem necessidade, gerando custo invisível.

Pipeline running on BigQuery Scheduled Queries — no version control, no explicit DAG, no PR review. A change to one table silently broke another. Tables on daily full-refresh with no reason, generating invisible cost.

// solução// solution

Migração para Dataform com ${ref()} em todas as dependências, DAG de 7 fases, workflow configs versionadas e scripts utilitários em Python para automatizar a conversão. Pattern A/B/C/D — quatro estágios de validação automatizada — garantindo paridade pré/pós em cada migração para incremental.

Migration to Dataform with ${ref()} on every dependency, a 7-phase DAG, versioned workflow configs, and Python utility scripts to automate the conversion. A/B/C/D pattern — four automated validation stages — guaranteeing pre/post parity on every incremental migration.

[04]   thecoffee.ml.case_04_store_clusters_v3
modelagem analítica  +  mlanalytical modeling  +  ml 2026

Health Score Multi-país & clusterização de lojas Multi-country Health Score & store clustering

// problema// problem

Rede internacional de franquias com lojas em contextos econômicos heterogêneos — moeda, paridade de poder de compra, maturidade de operação e perfil de público completamente diferentes. Comparar "loja A vs. loja B" em valores absolutos era enganoso, e a diretoria precisava de ranking ajustado e segmentação acionável.

International franchise network with stores in heterogeneous economic contexts — different currency, purchasing power parity, operational maturity, and customer profile. Comparing "store A vs. store B" in absolute terms was misleading, and the board needed an adjusted ranking and actionable segmentation.

// solução// solution

Modelo de health score ponderado em SQL nativo BigQuery, combinando performance, atingimento de meta, score digital e maturidade. Tags semânticas automáticas: store_maturity (Ramp-Up / Jovem / Madura / Consolidada), label_daypart, label_daytype e ppp_cluster (Tier 1–4 por paridade de poder de compra).

Weighted health-score model in native BigQuery SQL, combining performance, target attainment, digital score, and maturity. Automatic semantic tags: store_maturity (Ramp-Up / Young / Mature / Established), label_daypart, label_daytype, and ppp_cluster (Tier 1–4 by purchasing power parity).

// impacto// impact

Ranking comparável entre lojas de países distintos. A diretoria de operações passou a discutir loja por cluster comportamental ("Madura, almoço, dia de semana") em vez de média global. Modelo consumido em Looker como single source para health score da rede.

Comparable ranking across stores in different countries. Operations leadership now discusses stores by behavioral cluster ("Mature, lunch, weekday") instead of global averages. Model consumed in Looker as the single source for the network's health score.
